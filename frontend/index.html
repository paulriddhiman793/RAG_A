Beyond Basic RAG

A basic RAG system operates like "Ctrl+F with embeddings"—it flattens your document into plain text and blind-searches it.


NexusRAG is a multimodal, citation-grounded architecture that does much more:

  • Typed Extraction: It doesn't flatten files. It preserves Figures, Tables, and Math Formulas as distinct structural elements.
  • Vision Routing: For figure questions, it uses Llama Vision models to physically "look" at the charts.
  • Hybrid Indexing: Combines Vector semantic search with BM25 exact-keyword matching and cross-encoder Reranking.
  • Hierarchical Context: Searches tiny optimized chunks, but feeds large "Parent Chunks" to the AI so nothing is taken out of context.
  • Hallucination Guard: Runs an NLI-check to mathematically guarantee that all cited claims are grounded in document facts.
  • Agentic Reasoning Loop: Instead of blindly searching once, the Agent "thinks", runs queries, reads tables, compares data, and searches again until the task is complete.

Performance Trade-offs

Why does NexusRAG take more time than standard implementations?

Ingestion Constraints

Instead of cheap split-by-word-count chunking, NexusRAG invokes advanced Unstructured parsing and Vision ML models. It decomposes the visual layout, performs optical character recognition on messy math formulas, extracts raw HTML tables, and builds dual Parent-Child Vector/BM25 indices. This is heavy upfront compute yielding extreme analytic power.

Retrieval & Synthesis Constraints

Standard RAG takes 1-2 seconds because it simply dumps a chunk of text to the LLM. Because NexusRAG orchestrates Hybrid Retrieval + Reranking + Agent Loops, you are waiting for the AI to *actively research* locally. Running query expansions, bouncing between tools, and invoking Vision/NLI models takes longer but produces vastly more accurate, robust, and verified answers.
